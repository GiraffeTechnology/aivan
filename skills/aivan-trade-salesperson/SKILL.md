## Metadata

```yaml
openclaw:
  pluginRequirements:
    - package: "@giraffetechnology/openclaw-aivan"
      version: ">=0.1.0"
  routing:
    intent: trade-sourcing
    triggers:
      - buyer inquiry received
      - supplier reply received
      - sourcing request
      - lead time check
      - supplier risk screening
```

## Required environment variables

| Variable | Required | Default | Description |
|---|---|---|---|
| `AIVAN_BASE_URL` | Yes | `http://127.0.0.1:8765` | URL of the local AIVAN server |
| `AIVAN_API_KEY` | No | *(none)* | Optional bearer token for AIVAN API auth |

## Operating boundaries

This skill routes events to the AIVAN plugin. The skill itself contains no business logic.

**This skill DOES:**
- Detect trade-salesperson intent in incoming messages
- Instruct OpenClaw to invoke `aivan.forwardEvent` for matching events
- Return AIVAN's structured results to the conversation

**This skill DOES NOT:**
- Store supplier data, buyer data, or conversation history
- Send any outbound message on its own
- Make final sourcing, pricing, credit, or compliance decisions
- Access marketplace or IM accounts directly
- Store or transmit any credentials

## Human approval requirement

Every outbound message generated by AIVAN is held as a pending draft. No message is sent until a human operator explicitly approves it via `aivan.approveDraft` or the AIVAN dashboard.

This skill cannot override or bypass that policy.

## Local-first data boundary

All trade data is stored locally by AIVAN in a SQLite database on the operator's machine. This skill does not receive, cache, or persist any buyer or supplier data.

## Credential boundaries

This skill reads only `AIVAN_BASE_URL` and `AIVAN_API_KEY` from the environment. It does not handle, store, or transmit marketplace passwords, IM session tokens, or LLM API keys.

## Mock mode

AIVAN supports full mock mode (`AIVAN_LLM_PROVIDER=mock`, `OPENCLAW_MOCK_MODE=true`). The skill works identically in mock mode; no live credentials are required for testing.
